## What is TSS?

TSS is a browser-native utility-first CSS engine. Add a single `<script>` tag to your page and use class names like `p-4`, `text-lg`, `bg-primary`, and `flex` — TSS generates the CSS at runtime with no build step, no configuration, and no overhead.

```html index.html
<!DOCTYPE html>
<html>
  <head>
    <script src="https://cdn.jsdelivr.net/npm/tss-css/dist/tss.min.js"></script>
  </head>
  <body>
    <div class="flex flex-col items-center p-8 gap-4">
      <h1 class="text-4xl font-bold text-heading">Hello, TSS</h1>
      <p class="text-body text-lg max-w-md text-center">
        Zero-config utility-first CSS with automatic light/dark mode.
      </p>
      <button class="px-6 py-3 bg-accent text-white rounded-lg cursor-pointer transition">
        Get Started
      </button>
    </div>
  </body>
</html>
```

## Key features

<CardGroup cols={2}>
  <Card title="Zero config" icon="bolt">
    Auto-initializes on `DOMContentLoaded`. No configuration files, no PostCSS, no Tailwind config.
  </Card>
  <Card title="Auto light/dark mode" icon="moon">
    Respects `prefers-color-scheme` automatically. Override with `.dark` or `.light` on `<html>`.
  </Card>
  <Card title="Runtime CSS generation" icon="gears">
    Uses `MutationObserver` to detect new elements and generate utility CSS on the fly.
  </Card>
  <Card title="Semantic color tokens" icon="swatchbook">
    21 semantic CSS custom properties (`--tss-primary`, `--tss-accent`, etc.) that switch with the theme.
  </Card>
  <Card title="Tailwind-compatible" icon="check">
    Familiar class naming: `p-4`, `mt-8`, `text-xl`, `flex-col`, `grid-cols-3`, `rounded-lg`, and more.
  </Card>
  <Card title="TypeScript support" icon="file-code">
    Ships with `.d.ts` declarations for full type safety when using the JavaScript API.
  </Card>
</CardGroup>

## How it works

TSS runs entirely in the browser:

<Steps>
  <Step title="Script loads">
    TSS inserts two `<style>` elements into `<head>`: one for theme CSS custom properties, one for generated utility rules.
  </Step>
  <Step title="DOM scan">
    On `DOMContentLoaded`, TSS scans every element's `classList` and generates CSS rules for each recognized utility class.
  </Step>
  <Step title="MutationObserver watches">
    After init, a `MutationObserver` monitors `childList` and `class` attribute changes so dynamically added elements are styled automatically.
  </Step>
  <Step title="Theme auto-switches">
    The theme layer uses `@media (prefers-color-scheme: dark)` and `html.dark` / `html.light` overrides, so colors adapt with zero JavaScript required.
  </Step>
</Steps>
