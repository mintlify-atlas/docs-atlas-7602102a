TSS is a runtime CSS engine that reads utility classes directly from your HTML and generates the corresponding CSS on the fly — no build step, no configuration file, no PostCSS pipeline required.

Add a single script tag and start writing classes like `flex`, `p-4`, `text-xl`, and `bg-primary`. TSS handles the rest, including full light and dark mode with semantic color tokens that adapt automatically.

## How it compares to Tailwind

| | TSS | Tailwind CSS |
|---|---|---|
| Setup | None — one script tag | Build tool + config file required |
| CSS generation | Runtime (MutationObserver) | Build-time (PostCSS / CLI) |
| Dark mode | Automatic, persisted to `localStorage` | Manual `dark:` variant prefix |
| Bundle size | ~5 kB minified | ~10 kB+ (purged production build) |
| Custom config | Not required | `tailwind.config.js` required |
| Semantic color tokens | Built-in (`bg-primary`, `text-heading`) | Requires manual theme extension |

Tailwind is purpose-built for production applications with a full build pipeline. TSS is designed for projects where you want utility-class ergonomics without any tooling overhead — prototypes, static sites, documentation, embedded widgets, and more.

## How TSS works

When the script loads, TSS runs `restoreThemePreference()` to read the saved `tss-theme` value from `localStorage` before the page renders, preventing a flash of the wrong theme.

Once `DOMContentLoaded` fires, TSS:

1. Creates a `<style id="tss-theme">` element containing the full set of CSS custom property definitions for both light and dark mode.
2. Creates a `<style id="tss-utilities">` element where generated utility rules are appended as they are needed.
3. Scans `document.body` for all existing class names and generates CSS for any recognized utility classes.
4. Starts a `MutationObserver` that watches for `childList` changes (subtree) and `class` attribute mutations. When new nodes or class names appear, TSS parses them and appends any missing rules to `#tss-utilities`.

Because rules are only written once and reused thereafter, the generated stylesheet stays compact even on large pages.
